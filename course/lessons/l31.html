Module 3 · Admin-Assisted User Recovery

Password Resets from the Admin Console

The most common help-desk request you'll field. A few clicks in the Admin console, with one big caveat for SSO districts.

Where to do it

Reset a password from Admin console → Directory → Users. Hover the user's row and choose Reset password, or open the user's account page and use the same action there.

The reset workflow

  1. Go to Directory → Users and locate the user (search by name or email).
  2. Hover the row → Reset password (or open the account page first).
  3. Choose Automatically generate a password — recommended for compromised accounts — or Create password to type a specific one.
  4. Leave Ask the user to change their password checked to require a change at next sign-in (recommended).
  5. Click Reset.
  6. Optionally email the new password or a reset link to the user (or a recovery address).

Which option, when

OptionUse it for
Automatically generate a passwordCompromised or unknown-state accounts — you never handle a chosen secret, and a strong random value is created.
Create passwordRoutine help-desk resets where you'll read a temporary password to the user. Pair it with require-change-at-next-sign-in.

The Ask the user to change their password checkbox maps to require-change-at-next-sign-in. Keep it on so the temporary value you set never becomes the user's standing password.

Hard constraint — SSO and Password SyncIf your district uses third-party SSO or Google Cloud Directory Sync Password Sync, this Admin-console reset does not work. The password lives in your external identity provider (IdP) or Active Directory, not in Google. For those orgs, reset the password in the IdP — the Google-side reset will have no effect.
Verify in your consoleGoogle relabels actions and moves buttons periodically. Confirm the exact wording of Reset password, the password options, and the change-at-next-sign-in checkbox in your own Directory → Users before documenting it for your help desk. Note: Google is migrating Admin Help from support.google.com/a/… to knowledge.workspace.google.com/admin/…; old links redirect.
Director takeawayFor password-managed Google accounts, the Admin console is your one-stop reset. For any account fronted by an external IdP or Password Sync, the Admin console is the wrong tool — route those resets to the IdP and make sure your tier-1 staff know which population is which.
